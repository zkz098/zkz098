# QUIC and HTTP/3

# What is QUIC

QUIC is an abbreviation for Quick UDP Internet Connections and is currently divided into iQUIC and gQUIC.
QUIC is similar to TCP+TLS+HTTP/2 implemented at the kernel layer. But QUIC is based on UDP,
so it bypasses a lot of the cost of retrofitting TCP.
QUIC has several advantages over TLS+HTTP2：

  • First handshake 1-RTT
  • Improved TCP congestion
  • Forward error correction to reduce retransmissions.
  • Connections are smoothly migrated, and changes in network status do not affect connection disconnections.

# Why use QUIC

HTTP/2+TLS1.3v has been deployed on this website, which can theoretically implement a 0-RTT handshake.
DUE TO QUIC's optimization of weak network environments, QUIC was deployed.Because 4 times lost 1 transmission
Another secondary reason is the speed, and I personally prefer the new protocol.
