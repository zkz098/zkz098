    本文介绍了使用windbg工具分析蓝屏文件的过程。首先需要下载和安装windowsSDK，然后配置符号表，接着找到目标电脑的MEMORY.DMP文件并用windbg打开，输入`!analyze -v`进行深入分析，查看输出的内容来确定蓝屏原因。本文给出了一个实例，通过windbg分析得出是AMD显卡的驱动文件导致蓝屏。

    # 前言

    这次帮一个朋友解决蓝屏，但无法进行现场检测懒得出去，远程也看不到蓝屏代码
    只能使用 windbg 来分析内存文件检测蓝屏原因

    # 开始分析崩溃文件

    首先，你需要从目标电脑的 % SystemRoot%(一般为 C:/windows) 下找到 MEMORY.DMP 文件
    将其放到合适位置后，点击 File -> Open Crash Dump 选择崩溃文件

    然后输入 !analyze -v 对文件进行深入分析
    然后看输出的内容:

    输出的第一行为崩溃原因，一般和 win10 的蓝屏代码是一致的

    括号内的为 win7 版本，本案例就是 0x0000050

    这个代码的本意为崩溃原因和代码，并不只针对于 windows

    这两条代表 igdpmd64.sys 和 atikmpag.sys 驱动是概率存在问题的
    后来收到了对方发来的蓝屏图片，里面的关键内容如下:

    证实了是 igdpmd64.sys 导致的蓝屏，并且代码为 50
